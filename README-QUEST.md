# 🌸 Секретная миссия: Операция 8 Марта — Документация Проекта

Этот документ описывает структуру, логику и текущий прогресс интерактивного веб-квеста, созданного как поздравление для девушек (одногруппниц) и куратора на 8 Марта. 

Вы можете использовать этот файл как отправную точку при переносе проекта в другую IDE или для дальнейшей самостоятельной работы/доработки над логикой.

---

## 🎯 Цель проекта
Создать интерактивный сайт-квест («секретную миссию») длительностью около 10-15 минут, в котором участницы проходят цепочку из 7 небольших заданий, собирая по одной букве уникального кодового слова. 
Собрав слово полностью, девушка получает финальную инструкцию — отправить собранное кодовое слово "парням-организаторам" в обмен на подарок.

### Ключевые фичи:
- **Полная персонализация:** Участница выбирает своё имя на старте. Для каждой девушки подготовлен свой уникальный набор анонимных комплиментов, личное приветствие и **уникальное секретное слово из 7 букв/символов**.
- **Красочный UI:** Сайт использует современные плавные анимации, микро-взаимодействия и эффекты частиц (конфетти) для создания праздничного настроения. Тематика: "весенняя шпионская миссия".
- **Сброс прогресса:** Прогресс не теряется при перезагрузке страницы, но сбрасывается, если зайти под другим именем (через локальное хранилище).
- **Секреты (Пасхалки):** В проекте зашито много скрытых секретов для самых любопытных агентов (код Конами, секретная страница в URL, тройные клики, скрытые HTML-элементы).

## 🛠 Технический стек
1. **Фреймворк:** Next.js 15 (App Router)
2. **Язык:** TypeScript
3. **Стилевая библиотека:** Tailwind CSS 4
4. **UI-компоненты:** shadcn/ui (Radix UI) + Lucide React (иконки)
5. **Анимации:** framer-motion (появление, переходы, микро-взаимодействия) + canvas-confetti
6. **Стейт-менеджер:** Zustand (использует persist middleware для сохранения в localStorage)
7. **Drag & Drop:** @dnd-kit (для задания с фото-пазлом)

## 📁 Структура проекта

Наиболее важные директории и файлы для работы:

### Конфигурация данных (Содержимое)
- **`lib/data.ts`** — Главный файл с контентом! В нём лежат:
  - Списки имён (`NAMES`, `CURATOR_NAMES`).
  - Личные тайные кодовые слова (`PLAYER_SECRET_WORDS` — 10 слов по 7 символов).
  - Набор анонимных комплиментов от парней (`ANONYMOUS_COMPLIMENTS`). *Они пока заполнены заглушками (плейсхолдерами), нужно заменить на реальные фразы парней перед деплоем!*
  - Тексты поздравлений, общие фразы и элементы пасхалок.
- **`lib/tasks-config.ts`** — Конфигурация логики квеста (названия заданий, тексты-брифинги, тексты подсказок и правильные ответы на задания).
- **`lib/store.ts`** — Глобальный стор Zustand. Управляет прогрессом (какая буква собрана, на каком задании игрок, кто он).

### Экраны (Pages)
Система работает на App Router Next.js:
- **`app/page.tsx`** — Стартовая страница. Кнопка "Начать миссию", интерактивный фоновый генератор комплиментов и пасхалка.
- **`app/greeting/page.tsx`** — Страница инициализации. Работает как 3-шаговый flow:
   1. Выбор имени из списка (с автокомплитом).
   2. Страница персонального поздравления + анонимные карточки-комплименты от парней (переворачиваются по клику).
   3. Вводная часть миссии ("Тебе предстоит пройти 7 заданий...").
- **`app/quest/page.tsx`** — Само "тело" квеста. Отвечает за логику переключения заданий и отображение UI (через TaskRenderer).
- **`app/quest/final/page.tsx`** — Экран победы! Красочные анимации, демонстрация собранного индивидуального слова пользователя, ачивки (счетчик кликов и подсказок) и кнопка копирования слова.
- **`app/secret/page.tsx`** — Секретный маршрут, который используется для выполнения Задания №2.

### Компоненты (Components)
- **`components/quest/TaskRenderer.tsx`** — Главный дирижер заданий. Рисует заголовок и брифинг, динамически достает из словаря правильную букву для конкретной девочки и передает её внутрь нижележащего компонента-испытания.
- **`components/quest/tasks/...`** — Тут лежат компоненты всех 7 логических заданий. Если хочется изменить логику мини-игры (ускорить бабочку, поменять шифр), редактировать нужно именно здесь:
  - `CipherTask.tsx` (Шифр цезаря)
  - `SecretPageTask.tsx` (Секретный URL, роутится на `/secret`)
  - `PuzzleTask.tsx` (DnD сборка картинки)
  - `QuizTask.tsx` (Вопрос на знание численности группы)
  - `CatchButterflyTask.tsx` (Кликай по убегающему фрейму)
  - `FindFlowerTask.tsx` (Найти 1 цветок из множества без подсказок)
  - `SourceCodeTask.tsx` (Поиск HTML-комментария через F12 DevTools)

## 🚦 Как запустить и работать с кодом
1. Установите зависимости: `npm install`
2. Запустите в dev-режиме: `npm run dev`
3. Откройте в браузере `http://localhost:3000`
4. Для симуляции сброса и проверки квеста за разных людей: просто перейдите на главную страницу и выберите в списке другое имя. Состояние квеста (Zustand store) **автоматически очищается** (reset) для предыдущего игрока и создается с нуля.

## ✅ Что осталось сделать до релиза (Памятка)
1. Открыть `lib/data.ts` и заменить тексты-рыбу анонимных комплиментов (`ANONYMOUS_COMPLIMENTS`) на **реальные фразы, придуманные парнями**.
2. В том же файле (`lib/data.ts`) проверить `GROUP_SIZE`. Сейчас там стоит 26 агентов. Если численность группы другая — нужно поменять константу (влияет на задание №4).
3. Поменять изображение пазла в папке `public/images/puzzle.png` (если хотите вставить совместное фото или другую памятную картинку).
4. Опубликовать проект на Vercel или другом хостинге.
